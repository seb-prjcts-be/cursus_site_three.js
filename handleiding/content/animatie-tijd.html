Beweging werkt in three.js zoals in p5: elke frame een klein beetje veranderen. Alleen gebruik je geen frameCount maar een Timer - en die denkt in seconden.

Kernpunten
  • renderer.setAnimationLoop(fn) is je draw(): ongeveer 60 keer per seconde
  • THREE.Timer → update() + getElapsed() geeft de verstreken tijd in seconden
  • Tijd-gebaseerd animeren is framerate-onafhankelijk: op elk scherm even snel
  • Je p5-gereedschap blijft werken: Math.sin(), Math.cos() en lerp-denken

Van frameCount naar Timer

In p5 schreef je vaak rotateY(frameCount * 0.01). Dat kan hier ook (mesh.rotation.y += 0.01), maar er is een betere gewoonte: animeer op tijd.

const timer = new THREE.Timer();

renderer.setAnimationLoop(() => {
  timer.update();                     // intern bijwerken, één keer per frame
  const t = timer.getElapsed();       // seconden sinds de start

  mesh.rotation.y = t;                       // 1 radiaal per seconde
  mesh.position.y = Math.sin(t * 2) * 1.5;   // op en neer

  renderer.render(scene, camera);
});

Waarom beter? Een telefoon op 30 fps en een gamemonitor op 144 fps tonen nu exact dezelfde beweging. Met frame-tellen draait je sketch op de gamemonitor bijna vijf keer zo snel.

Vuistregel om te lezen: rotation.y = t betekent één radiaal per seconde, dus een volledige draai duurt zo'n 6,3 seconden. Wil je het dubbel zo snel? rotation.y = t * 2.

Een golf van kubussen

Het klassieke p5-patroon "een rij dingen, elk met een fase-verschil in sin()" werkt hier letterlijk hetzelfde. Het enige verschil: de kubussen bestaan al, je wijzigt alleen hun y-positie.

Weetje

Let op de regel met geom en mat: alle vijftien kubussen delen één geometrie en één materiaal. three.js tekent dat veel sneller dan vijftien aparte kopieën - en het is precies waarom geometry en material gescheiden zijn.

Soepel naar een doel: lerp

Het p5-patroon x = lerp(x, doelX, 0.1) bestaat ingebouwd op elke positie:

// Elke frame 10% dichter bij het doel - soepel afremmen
mesh.position.lerp(new THREE.Vector3(2, 1, 0), 0.1);

Posities in three.js zijn Vector3-objecten, de 3D-versie van p5.Vector. Veel methodes ken je al: add(), sub(), normalize(), dist()... je vector-kennis uit p5 verhuist gewoon mee.

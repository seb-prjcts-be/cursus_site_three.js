The live editors are great for learning, but real work happens in a file of your own. This page gives you the complete starter code and explains how to set up a three.js project on your own computer.

What you will learn
  • One index.html with an importmap is enough - no npm, no build tools
  • The importmap tells the browser where import 'three' should come from
  • You need a local server (file:// does not work with modules)
  • The starter code below contains the full ritual, including resize and OrbitControls

The starter code

Copy this as index.html into a folder of your own. This is the boilerplate that combines everything from the course - an empty canvas, ready to build on:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>My three.js sketch</title>
  <style> body { margin: 0; overflow: hidden; } </style>
  <script type="importmap">
  {
    "imports": {
      "three": "https://cdn.jsdelivr.net/npm/three@0.184.0/build/three.module.js",
      "three/addons/": "https://cdn.jsdelivr.net/npm/three@0.184.0/examples/jsm/"
    }
  }
  </script>
</head>
<body>
<script type="module">
import * as THREE from 'three';
import { OrbitControls } from 'three/addons/controls/OrbitControls.js';

// --- the ritual ---
const scene = new THREE.Scene();
scene.background = new THREE.Color(0x141414);

const camera = new THREE.PerspectiveCamera(60, innerWidth / innerHeight, 0.1, 1000);
camera.position.set(3, 2, 5);

const renderer = new THREE.WebGLRenderer({ antialias: true });
renderer.setSize(innerWidth, innerHeight);
document.body.appendChild(renderer.domElement);

const controls = new OrbitControls(camera, renderer.domElement);
controls.enableDamping = true;

// --- light ---
const sun = new THREE.DirectionalLight(0xffffff, 3);
sun.position.set(5, 5, 5);
scene.add(sun, new THREE.AmbientLight(0xffffff, 0.3));

// --- your scene starts here ---
const cube = new THREE.Mesh(
  new THREE.BoxGeometry(1, 1, 1),
  new THREE.MeshStandardMaterial({ color: 0xff007d })
);
scene.add(cube);

// --- resize ---
addEventListener('resize', () => {
  camera.aspect = innerWidth / innerHeight;
  camera.updateProjectionMatrix();
  renderer.setSize(innerWidth, innerHeight);
});

// --- the loop ---
const timer = new THREE.Timer();
renderer.setAnimationLoop(() => {
  timer.update();
  const t = timer.getElapsed();

  cube.rotation.y = t;

  controls.update();
  renderer.render(scene, camera);
});
</script>
</body>
</html>

Working on the school computers or at home next to this course? Then replace the two CDN addresses with the local clone: "../three.js/build/three.module.js" and "../three.js/examples/jsm/" - your sketch will then work without internet too.

Starting a server

ES modules do not load from file:// - the same limitation as loadImage() in p5. Three ways, pick what suits your setup:

  1. VS Code: Live Server

    Install the "Live Server" extension, right-click your index.html and choose "Open with Live Server". Reloads automatically on every change - the closest to the p5 editor feeling.

  2. Python (often already on your computer)

    Open a terminal in your project folder and type python -m http.server. Browse to http://localhost:8000.

  3. A folder under an existing server

    Already running Apache or XAMPP (like on the Lab44 machines)? Put your project folder in htdocs and browse to localhost/yourfolder/.

Versions: why that @0.184.0 in the importmap?

three.js changes every month. Pinning a version number means your sketch will still work exactly the same a year from now. Leave it out and you get "the latest" - and an update can break your sketch while you changed nothing.

Watch out

Older tutorials on the internet load three.js with a plain script tag without an importmap, or use THREE as a global variable without an import. That is the outdated way - if you recognise it, know that you first translate such code to the module pattern of this course.

Did you know

For bigger projects developers switch to npm and a bundler (Vite is the most popular). That gives faster reload times and tidy imports, but changes nothing about the three.js code itself. Everything from this course moves over one-to-one.

In p5 a shape is one command: box(100). In three.js every shape consists of two parts: a geometry (the shape) and a material (the appearance). Together they become a mesh.

What you will learn
  • Mesh = Geometry + Material - shape and appearance are separated
  • Geometries: BoxGeometry, SphereGeometry, CylinderGeometry, TorusGeometry, PlaneGeometry, ...
  • MeshStandardMaterial responds to light, MeshBasicMaterial doesn't (like a flat fill)
  • You may reuse one geometry or material for many meshes - that's free performance

Shape + appearance = mesh

The p5 line fill(255, 0, 125); box(100); becomes in three.js:

const geometry = new THREE.BoxGeometry(1, 1, 1);  // the shape
const material = new THREE.MeshStandardMaterial({ color: 0xff007d });  // the appearance
const mesh = new THREE.Mesh(geometry, material);   // together: a thing in the world
scene.add(mesh);

Why that split? Because you can reuse parts. A hundred spheres with the same shape but different colours: one geometry, a hundred materials. A hundred identical marbles: one geometry, one material, a hundred meshes.

The shape library

The names look a lot like what you know from p5 WEBGL mode:

  • BoxGeometry(w, h, d) - p5: box()
  • SphereGeometry(radius) - p5: sphere()
  • CylinderGeometry(radiusTop, radiusBottom, height) - p5: cylinder(), but with two radii: use it to make a cone
  • TorusGeometry(radius, tube) - p5: torus()
  • PlaneGeometry(w, h) - p5: plane()
  • TorusKnotGeometry(...) - doesn't exist in p5, and is far too beautiful to ignore

Materials: the new fill()

In p5 you choose a drawing state: fill, stroke, normalMaterial. In three.js you choose a material type per object:

  • MeshBasicMaterial - flat colour, ignores light. Like fill() without lights(). Handy for debugging: this material is never "accidentally black".
  • MeshNormalMaterial - rainbow colours based on the direction of each face. Identical to p5's normalMaterial().
  • MeshStandardMaterial - physically realistic, responds to light. With roughness and metalness, each from 0 to 1. This is your default choice.
new THREE.MeshStandardMaterial({
  color: 0xff007d,
  roughness: 0.3,   // 0 = mirror smooth, 1 = matte
  metalness: 0.8    // 0 = plastic, 1 = metal
});

Object black and you don't know why? Temporarily replace the material with MeshNormalMaterial. If you can see the object then, your problem is the light, not the geometry.

Did you know

Under the bonnet, a geometry is simply a list of triangles. SphereGeometry(1, 32, 16) builds a sphere from 32 by 16 segments. Fewer segments = more angular and faster, more = rounder and slower. That's the same as the detail parameters of p5's sphere().

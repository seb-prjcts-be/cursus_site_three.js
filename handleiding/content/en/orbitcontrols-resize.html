With OrbitControls you use the mouse to orbit around your scene - the same feel as orbitControl() in p5. And because three.js does nothing automatically, you'll also learn here how to make your scene scale along with the window.

What you will learn
  • OrbitControls is an addon: you import it from three/addons/
  • Drag = rotate, scroll = zoom, right-drag = pan
  • enableDamping gives a smooth coasting effect, but requires controls.update() in the loop
  • On window resize you update camera.aspect and renderer.setSize() yourself

Addons: the second import

The core of three.js keeps itself small. A lot of the good stuff lives in the addons folder: OrbitControls, model loaders, effects. You recognise them by the second import line:

import * as THREE from 'three';
import { OrbitControls } from 'three/addons/controls/OrbitControls.js';

// after creating the camera and renderer:
const controls = new OrbitControls(camera, renderer.domElement);
controls.enableDamping = true;   // smooth coasting

renderer.setAnimationLoop(() => {
  controls.update();             // needed for damping
  renderer.render(scene, camera);
});

Try it: drag, scroll, pan

Handy options to try in the editor: controls.autoRotate = true (the camera rotates by itself, great for presentations), controls.minDistance and maxDistance (limit the zooming), and controls.enablePan = false (switch off panning).

Scaling along with the window

p5 had windowResized() and resizeCanvas(). In three.js you write that reflex yourself - three lines that belong in every fullscreen sketch:

addEventListener('resize', () => {
  camera.aspect = innerWidth / innerHeight;
  camera.updateProjectionMatrix();           // make the camera recalculate
  renderer.setSize(innerWidth, innerHeight);
});
Watch out

If you forget camera.updateProjectionMatrix(), the image stays stretched after resizing. The camera remembers its settings until you explicitly say: recalculate. This is the most forgotten line in three.js.

Did you know

GridHelper from the example is one of many helpers: AxesHelper shows the x, y and z axes as red, green and blue lines, and is worth its weight in gold when you no longer know which way is "forward". You'll find more helpers in the lesson Debugging & speed.

A constrained camera

Make sure the viewer can't dive below the floor and can't zoom out infinitely far: combine maxPolarAngle, minDistance and maxDistance.

Hint
controls.maxPolarAngle = Math.PI / 2.1;  // just above the floor
controls.minDistance = 3;
controls.maxDistance = 15;

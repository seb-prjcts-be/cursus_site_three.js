three.js is the best-known JavaScript library for 3D in the browser. Where p5.js was made for sketching quickly, three.js was made for building worlds.

What you will learn
  • three.js was started in 2010 by Ricardo Cabello, known online as mrdoob
  • It builds on top of WebGL, the same technology as p5.js in WEBGL mode
  • p5.js = quick sketching, three.js = fully-fledged 3D worlds with materials, light and cameras
  • You write three.js as an ES module: with import, not with one big global API

What is three.js?

three.js is an open source library that makes 3D graphics in the browser achievable. Programming WebGL directly takes hundreds of lines of code for a single triangle. three.js hides that work behind understandable building blocks: a scene, a camera, shapes, materials and light.

You will find three.js everywhere: in data visualisations, product configurators, music videos, games and digital art. Many award-winning websites run on three.js.

Did you know

Ricardo Cabello started three.js in 2010, initially in ActionScript for Flash. The name comes from "3D" - three dimensions. The library is still updated every month, with releases that carry r-numbers (r150, r160, ...) instead of regular version numbers.

When p5.js, when three.js?

Both libraries can do 3D. The difference lies in what they were made for:

  • p5.js - sketching an idea quickly, generative art, 2D, teaching, prototypes. 3D is possible with WEBGL mode, but stays limited.
  • three.js - serious 3D: realistic materials, shadows, loading 3D models, large scenes with hundreds of objects, VR.

Rule of thumb: if your idea starts with drawing, pick p5. If it starts with a world or object in 3D, pick three.js.

You don't have to choose forever. The thinking patterns you learned in p5 - a setup, an animation loop, transformations, sin() and cos() for movement - you simply reuse in three.js.

What does three.js code look like?

Unlike p5, three.js has no magic functions like setup() and draw() that exist automatically. You import the library as a module and build everything up yourself:

import * as THREE from 'three';

const scene = new THREE.Scene();
const camera = new THREE.PerspectiveCamera(75, innerWidth / innerHeight, 0.1, 1000);
const renderer = new THREE.WebGLRenderer();

That feels more formal than p5 at first. The payoff: you understand exactly which parts exist, because you made them yourself. In Your first scene you build up this ritual step by step.

Watch out

Because three.js is an ES module, your sketch won't work if you open the HTML file straight from your disk (file://). You need a local server - just like p5 with loadImage(). On this site that's already taken care of: the live editors below just work.

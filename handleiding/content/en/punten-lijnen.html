Not everything is a solid object. With Points you draw thousands of floating particles in one go - starry skies, snow, data clouds. This is the three.js territory of your p5 particle systems, but with tens of thousands at once.

What you will learn
  • A BufferGeometry + list of positions + PointsMaterial = a point cloud
  • Positions live in one flat Float32Array: [x1, y1, z1, x2, y2, z2, ...]
  • One Points object with 10,000 points is much faster than 10,000 meshes
  • The same idea exists for lines: Line and LineSegments

From Particle class to point cloud

In p5 you wrote a Particle class and an array with a hundred instances. From a few thousand particles onwards, that got slow. three.js takes a different approach: all positions go to the graphics card in one package:

const count = 5000;
const positions = new Float32Array(count * 3);  // x, y, z per point

for (let i = 0; i < count; i++) {
  positions[i * 3]     = (Math.random() - 0.5) * 20;  // x
  positions[i * 3 + 1] = (Math.random() - 0.5) * 20;  // y
  positions[i * 3 + 2] = (Math.random() - 0.5) * 20;  // z
}

const geom = new THREE.BufferGeometry();
geom.setAttribute('position', new THREE.BufferAttribute(positions, 3));

const cloud = new THREE.Points(geom, new THREE.PointsMaterial({
  color: 0xff007d,
  size: 0.05
}));
scene.add(cloud);
Did you know

Every geometry you have used so far - BoxGeometry, SphereGeometry - is also just a BufferGeometry with such a position list, plus triangle information. So here you are looking under the bonnet of everything you already know.

Moving points individually

If you want to move each point separately (like your particles in p5), you change the values in the position array and set one flag:

const pos = geom.attributes.position;
for (let i = 0; i < pos.count; i++) {
  pos.setY(i, pos.getY(i) + Math.sin(t + i) * 0.002);
}
pos.needsUpdate = true;   // tell the graphics card!
Watch out

Forget needsUpdate = true and nothing moves - the graphics card still has the old copy. This is THE classic BufferGeometry pitfall, so memorise it now.

Lines work identically: same geometry, different material and object - new THREE.Line(geom, new THREE.LineBasicMaterial({color: 0xffffff})) connects the points in order. Use it to draw trajectories, graphs and wire sculptures.

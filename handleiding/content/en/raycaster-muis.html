In p5 you asked "is the mouse inside this circle?" with a bit of distance maths. In 3D that's trickier: the mouse lives on a flat screen, your objects in depth. The Raycaster solves that: it shoots a ray from the camera through the mouse, and tells you which object it hits.

What you will learn
  • You convert mouse coordinates to the range -1 to +1 (normalised)
  • raycaster.setFromCamera(mouse, camera) aims the ray
  • intersectObjects(...) returns the objects that were hit, closest first
  • This is called picking - the 3D answer to "is mouseX inside my shape?"

The pattern

Three fixed steps, which come back almost literally in every interactive scene:

const raycaster = new THREE.Raycaster();
const mouse = new THREE.Vector2();

addEventListener('pointermove', (event) => {
  // 1. Convert the mouse to -1..+1 (bottom-left to top-right)
  mouse.x =  (event.clientX / innerWidth)  * 2 - 1;
  mouse.y = -(event.clientY / innerHeight) * 2 + 1;
});

renderer.setAnimationLoop(() => {
  // 2. Aim the ray from the camera through the mouse
  raycaster.setFromCamera(mouse, camera);

  // 3. Ask what it hits
  const hits = raycaster.intersectObjects(scene.children);
  if (hits.length > 0) {
    hits[0].object.material.color.set(0xff007d);
  }

  renderer.render(scene, camera);
});

Try it: move over the cubes

hits[0] is always the closest object - after all, the ray can pierce several cubes one behind the other. Want to click instead of hover? Use a pointerdown listener and do the raycast in there.

Watch out

In this example every cube gets its own material, while on the Animation & time page you learned to share materials. That is deliberate: shared material = shared colour. If you want to colour one object individually, it needs its own material.

Click instead of hover

Make cubes stay pink once you click them (pointerdown), and white again on a second click. Store the state per cube in userData.

Hint
addEventListener('pointerdown', () => {

  if (hits.length > 0) {
    const k = hits[0].object;
    k.userData.on = !k.userData.on;

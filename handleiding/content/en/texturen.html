A texture is an image wrapped around a 3D shape. The TextureLoader is the three.js version of loadImage() - and you stick the result straight onto a material.

What you will learn
  • new THREE.TextureLoader().load(path) loads an image as a texture
  • You attach the texture to the material via the map property
  • Loading is asynchronous, but three.js shows the object right away and fills in the texture as soon as it arrives - no preload() needed
  • Every geometry has built-in UV coordinates that determine how the image wraps around the shape

From loadImage to TextureLoader

// p5
let img;
function preload() { img = loadImage('stone.jpg'); }

// three.js
const texture = new THREE.TextureLoader().load('stone.jpg');
const material = new THREE.MeshStandardMaterial({ map: texture });

No preload hassle: the renderer just keeps drawing and the texture appears as soon as it has loaded. The property is called map because there are many more "maps": roughnessMap, normalMap, bumpMap - all images that control a material property per pixel.

Try it: the UV test grid

This example loads the official UV test grid from your local three.js clone. Each square shows where that part of the image ends up on the shape. Try swapping the geometry: replace the cube with a SphereGeometry(1.5) or TorusKnotGeometry(1, 0.4).

The line texture.colorSpace = THREE.SRGBColorSpace should always be there for colour images. Forget it, and everything looks just a bit washed out and wrong - subtle enough to keep you searching for a long time.

Did you know

UV coordinates are called that because x and y were already taken by 3D space. U runs horizontally through the image, V vertically, both from 0 to 1. The test grid with letters and numbers exists precisely so you can see which (U, V) square lands where.

Your own images

Anything loadImage() could load, TextureLoader can too: photos, your own drawings, even a p5 export. Watch out for two things:

  • The image has to be reachable via the same server (no file://, and other people's servers often block - the same CORS story as with p5).
  • Dimensions in powers of two (256, 512, 1024) render the most efficiently.

Your local clone has a whole folder to play with: three.js/examples/textures/ - from brick to lava rock.

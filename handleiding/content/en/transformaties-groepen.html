No translate(), no push() and pop(): in three.js every object carries its own position, rotation and scale. And for hierarchy - things that move along with other things - you use a Group.

What you will learn
  • Every object has .position, .rotation and .scale - all three with x, y and z
  • There is no matrix stack: transformations are properties, not commands
  • A Group is the new push()/pop(): children move along with their parent
  • The scene itself is the root of that tree - hence the name scene graph

Properties instead of commands

Compare how you place and rotate an object in both worlds:

// p5: commands that change the drawing state
push();
translate(2, 0, 0);
rotateY(0.8);
box(1);
pop();

// three.js: properties of the object itself
cube.position.x = 2;
cube.rotation.y = 0.8;

Handy shortcuts for writing faster:

mesh.position.set(2, 1, 0);   // x, y, z in one go
mesh.scale.setScalar(2);      // everything twice as big
mesh.rotation.y += 0.01;      // rotating in the loop

Groups: the new push()/pop()

In p5, push()/pop() isolated transformations. In three.js you build a tree instead: a Group is an invisible container, and everything inside it moves along.

Classic example: a planet with a moon. If the group rotates, the moon orbits the planet:

Try this in the editor: add a second moon with a different position.x, or put the whole system inside yet another group that you tilt with system2.rotation.z. Groups within groups - that's how you build a solar system, a robot arm, a bicycle.

Thinking in the scene graph

The scene is a tree. The scene is the trunk, groups are branches, meshes are leaves. Every position you set is relative to the parent:

scene
└── system (Group)      rotation.y rotates everything
    ├── planet (Mesh)   at (0, 0, 0) inside the group
    └── moon (Mesh)     at (2.5, 0, 0) inside the group

So the moon is not "at x = 2.5 in the world", but "at x = 2.5 relative to the system". Exactly what nested push()/pop() blocks did, but now you can draw it as a tree and it keeps existing between frames.

Did you know

Everything in three.js inherits from the same base class Object3D: meshes, groups, cameras and even lights. That's why you can also put a camera or a lamp inside a group and let it move along - a camera on a rollercoaster is just scene graph work.

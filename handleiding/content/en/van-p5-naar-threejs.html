Everything you learned in p5.js also exists in three.js - it just has a different name and works slightly differently. This page is your dictionary.

What you will learn
  • p5 is immediate mode: every frame you draw everything again
  • three.js is retained mode: objects exist permanently in a scene graph
  • In the animation loop you don't draw anything - you change properties of existing objects
  • The translation table below links every p5 concept to its three.js equivalent

The big difference: drawing versus existing

In p5, background() wipes the canvas and you then draw everything again. A circle only exists at the moment circle() is executed:

// p5: the circle only "exists" during this command
function draw() {
  background(240);
  circle(x, y, 60);
}

In three.js you create objects once. They keep existing in the scene, and in the loop you only change their properties:

// three.js: the sphere EXISTS, every frame you move it
const sphere = new THREE.Mesh(geometry, material);
scene.add(sphere);

renderer.setAnimationLoop(() => {
  sphere.position.x += 0.01;      // change a property
  renderer.render(scene, camera); // draw the tree
});

Do you know classes in p5? Then you already know this thinking pattern. A class Ball with properties x and y that you update in draw() - that is exactly how every three.js object works. three.js has that pattern built in.

Pitfalls for p5 hands

  1. Don't create objects in the loop

    new THREE.SphereGeometry() in the animation loop is the three.js version of loadImage() in draw(): your memory fills up. Create objects once, then only change their properties.

  2. Y points up

    Just like in p5 WEBGL mode, but unlike p5 2D. Positive y is upwards, and (0, 0, 0) lies in the middle of the world.

  3. Colours are hex numbers

    0xff007d instead of (255, 0, 125). If you still want to write p5-style, new THREE.Color('rgb(255, 0, 125)') works too.

  4. Angles are always radians

    There is no angleMode(DEGREES). Use THREE.MathUtils.degToRad(90) if you want to think in degrees.

  5. Without light everything is black

    The standard material responds to light, like p5 shapes after lights(). No light added? Black object. More on this in Light.

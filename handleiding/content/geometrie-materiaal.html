In p5 is een vorm één commando: box(100). In three.js bestaat elke vorm uit twee delen: een geometrie (de vorm) en een materiaal (het uiterlijk). Samen worden ze een mesh.

Kernpunten
  • Mesh = Geometry + Material - vorm en uiterlijk zijn gescheiden
  • Geometrieën: BoxGeometry, SphereGeometry, CylinderGeometry, TorusGeometry, PlaneGeometry, ...
  • MeshStandardMaterial reageert op licht, MeshBasicMaterial niet (zoals een platte fill)
  • Eén geometrie of materiaal mag je hergebruiken voor veel meshes - dat is gratis performance

Vorm + uiterlijk = mesh

De p5-regel fill(255, 0, 125); box(100); wordt in three.js:

const geometry = new THREE.BoxGeometry(1, 1, 1);  // de vorm
const material = new THREE.MeshStandardMaterial({ color: 0xff007d });  // het uiterlijk
const mesh = new THREE.Mesh(geometry, material);   // samen: een ding in de wereld
scene.add(mesh);

Waarom die splitsing? Omdat je delen kunt hergebruiken. Honderd bollen met dezelfde vorm maar andere kleuren: één geometrie, honderd materialen. Honderd identieke knikkers: één geometrie, één materiaal, honderd meshes.

De vormen-bibliotheek

De namen lijken sterk op wat je kent uit p5 WEBGL-mode:

  • BoxGeometry(b, h, d) - p5: box()
  • SphereGeometry(straal) - p5: sphere()
  • CylinderGeometry(stralBoven, straalOnder, hoogte) - p5: cylinder(), maar met twee stralen: maak er een kegel mee
  • TorusGeometry(straal, buisdikte) - p5: torus()
  • PlaneGeometry(b, h) - p5: plane()
  • TorusKnotGeometry(...) - bestaat niet in p5, en is veel te mooi om te negeren

Materialen: de nieuwe fill()

In p5 kies je een tekenstaat: fill, stroke, normalMaterial. In three.js kies je per object een materiaal-type:

  • MeshBasicMaterial - platte kleur, negeert licht. Zoals fill() zonder lights(). Handig om te debuggen: dit materiaal is nooit "per ongeluk zwart".
  • MeshNormalMaterial - regenboogkleuren op basis van de richting van elk vlak. Identiek aan p5's normalMaterial().
  • MeshStandardMaterial - fysisch realistisch, reageert op licht. Met roughness (ruwheid) en metalness (metaalglans), elk van 0 tot 1. Dit is je standaardkeuze.
new THREE.MeshStandardMaterial({
  color: 0xff007d,
  roughness: 0.3,   // 0 = spiegelglad, 1 = mat
  metalness: 0.8    // 0 = plastic, 1 = metaal
});

Object zwart en je weet niet waarom? Vervang het materiaal tijdelijk door MeshNormalMaterial. Zie je het object dan wel, dan is je probleem het licht, niet de geometrie.

Weetje

Een geometrie is onder de motorkap gewoon een lijst driehoeken. SphereGeometry(1, 32, 16) maakt een bol uit 32 bij 16 segmenten. Minder segmenten = hoekiger en sneller, meer = ronder en trager. Dat is hetzelfde als detail-parameters bij p5's sphere().

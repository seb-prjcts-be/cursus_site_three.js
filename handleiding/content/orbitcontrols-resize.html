Met OrbitControls draai je met de muis rond je scene - hetzelfde gevoel als orbitControl() in p5. En omdat three.js niets vanzelf doet, leer je hier ook je scene laten meeschalen met het venster.

Kernpunten
  • OrbitControls is een addon: je importeert hem uit three/addons/
  • Slepen = draaien, scrollen = zoomen, rechts slepen = verschuiven
  • enableDamping geeft een soepel uitdraai-effect, maar vraagt controls.update() in de loop
  • Bij venster-resize werk je zelf camera.aspect en renderer.setSize() bij

Addons: de tweede import

De kern van three.js houdt zichzelf klein. Veel goeds leeft in de addons-map: OrbitControls, model-loaders, effecten. Je herkent ze aan de tweede import-regel:

import * as THREE from 'three';
import { OrbitControls } from 'three/addons/controls/OrbitControls.js';

// na het maken van camera en renderer:
const controls = new OrbitControls(camera, renderer.domElement);
controls.enableDamping = true;   // soepel uitdraaien

renderer.setAnimationLoop(() => {
  controls.update();             // nodig voor damping
  renderer.render(scene, camera);
});

Probeer het: sleep, scroll, verschuif

Handige opties om in de editor te proberen: controls.autoRotate = true (de camera draait vanzelf, mooi voor presentaties), controls.minDistance en maxDistance (beperk het zoomen), en controls.enablePan = false (verschuiven uitzetten).

Meeschalen met het venster

p5 had windowResized() en resizeCanvas(). In three.js schrijf je die reflex zelf - drie regels die in elke fullscreen-sketch thuishoren:

addEventListener('resize', () => {
  camera.aspect = innerWidth / innerHeight;
  camera.updateProjectionMatrix();           // camera laten herrekenen
  renderer.setSize(innerWidth, innerHeight);
});
Let op

Vergeet je camera.updateProjectionMatrix(), dan blijft het beeld uitgerokken na het resizen. De camera onthoudt zijn instellingen tot je expliciet zegt: herbereken. Dit is de meest vergeten regel van three.js.

Weetje

GridHelper uit het voorbeeld is één van de vele helpers: AxesHelper toont de x-, y- en z-as als rode, groene en blauwe lijnen, en is goud waard als je niet meer weet welke kant "voor" is. Meer helpers vind je in de les Debuggen & snelheid.

Vanzelf draaien

Zet controls.autoRotate = true en speel met controls.autoRotateSpeed. Sleep tijdens het draaien: jij wint altijd van de autorotatie.

Variëren

Breid een voorbeeld uit met eigen aanpassingen.

Begrensde camera

Zorg dat de kijker niet onder de vloer kan duiken en niet oneindig ver kan wegzoomen: combineer maxPolarAngle, minDistance en maxDistance.

Hint
controls.maxPolarAngle = Math.PI / 2.1;  // net boven de vloer
controls.minDistance = 3;
controls.maxDistance = 15;

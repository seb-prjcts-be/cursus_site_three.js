three.js is de bekendste JavaScript bibliotheek voor 3D in de browser. Waar p5.js gemaakt is om snel te schetsen, is three.js gemaakt om werelden te bouwen.

Kernpunten
  • three.js is gestart in 2010 door Ricardo Cabello, online bekend als mrdoob
  • Het bouwt bovenop WebGL, dezelfde techniek als p5.js in WEBGL-mode
  • p5.js = snel schetsen, three.js = volwaardige 3D-werelden met materialen, licht en camera's
  • Je schrijft three.js als ES-module: met import, niet met een grote globale API

Wat is three.js?

three.js is een open source bibliotheek die 3D graphics in de browser haalbaar maakt. Rechtstreeks WebGL programmeren is honderden regels code voor één driehoek. three.js verstopt dat werk achter begrijpelijke bouwstenen: een scene, een camera, vormen, materialen en licht.

Je vindt three.js overal: in datavisualisaties, productconfigurators, muziekvideo's, games en digitale kunst. Veel award-winnende websites draaien op three.js.

Weetje

Ricardo Cabello begon three.js in 2010, eerst nog in ActionScript voor Flash. De naam komt van "3D" - drie dimensies. De bibliotheek wordt tot vandaag elke maand bijgewerkt, met releases die r-nummers dragen (r150, r160, ...) in plaats van gewone versienummers.

Wanneer p5.js, wanneer three.js?

Beide bibliotheken kunnen 3D. Het verschil zit in waar ze voor gemaakt zijn:

  • p5.js - snel een idee schetsen, generatieve kunst, 2D, lesgeven, prototypes. 3D kan met WEBGL-mode, maar blijft beperkt.
  • three.js - serieuze 3D: realistische materialen, schaduwen, 3D-modellen laden, grote scenes met honderden objecten, VR.

Vuistregel: begint je idee bij tekenen, kies p5. Begint het bij een wereld of object in 3D, kies three.js.

Je hoeft niet te kiezen voor altijd. De denkpatronen die je in p5 leerde - een setup, een animatieloop, transformaties, sin() en cos() voor beweging - gebruik je in three.js gewoon opnieuw.

Hoe ziet three.js-code eruit?

Anders dan p5 heeft three.js geen magische functies zoals setup() en draw() die vanzelf bestaan. Je importeert de bibliotheek als module en bouwt alles zelf op:

import * as THREE from 'three';

const scene = new THREE.Scene();
const camera = new THREE.PerspectiveCamera(75, innerWidth / innerHeight, 0.1, 1000);
const renderer = new THREE.WebGLRenderer();

Dat voelt eerst formeler dan p5. De winst: je begrijpt precies welke onderdelen er bestaan, want je hebt ze zelf gemaakt. In Je eerste scene bouw je dit ritueel stap voor stap op.

Let op

Niet alles is een massief object. Met Points teken je duizenden zwevende deeltjes in één klap - sterrenhemels, sneeuw, datawolken. Dit is het three.js-territorium van je p5-particle-systemen, maar dan met tienduizenden tegelijk.

Kernpunten
  • Een BufferGeometry + lijst van posities + PointsMaterial = een puntenwolk
  • Posities leven in één platte Float32Array: [x1, y1, z1, x2, y2, z2, ...]
  • Eén Points-object met 10.000 punten is véél sneller dan 10.000 meshes
  • Voor lijnen bestaat hetzelfde idee: Line en LineSegments

Van Particle-class naar puntenwolk

In p5 maakte je een class Particle en een array met honderd instanties. Vanaf een paar duizend deeltjes werd dat traag. three.js pakt het anders aan: álle posities in één pakket naar de videokaart:

const aantal = 5000;
const posities = new Float32Array(aantal * 3);  // x, y, z per punt

for (let i = 0; i < aantal; i++) {
  posities[i * 3]     = (Math.random() - 0.5) * 20;  // x
  posities[i * 3 + 1] = (Math.random() - 0.5) * 20;  // y
  posities[i * 3 + 2] = (Math.random() - 0.5) * 20;  // z
}

const geom = new THREE.BufferGeometry();
geom.setAttribute('position', new THREE.BufferAttribute(posities, 3));

const wolk = new THREE.Points(geom, new THREE.PointsMaterial({
  color: 0xff007d,
  size: 0.05
}));
scene.add(wolk);
Weetje

Elke geometrie die je tot nu toe gebruikte - BoxGeometry, SphereGeometry - is óók gewoon een BufferGeometry met zo'n positie-lijst, plus driehoek-informatie. Je kijkt hier dus onder de motorkap van alles wat je al kent.

Punten individueel bewegen

Wil je elk punt apart laten bewegen (zoals je particles in p5), dan wijzig je de waarden in de positie-array en zet je één vlag:

const pos = geom.attributes.position;
for (let i = 0; i < pos.count; i++) {
  pos.setY(i, pos.getY(i) + Math.sin(t + i) * 0.002);
}
pos.needsUpdate = true;   // zeg het tegen de videokaart!
Let op

Vergeet je needsUpdate = true, dan beweegt er niets - de videokaart heeft nog de oude kopie. Dit is dé instinker van BufferGeometry, onthoud hem nu alvast.

Lijnen werken identiek: zelfde geometrie, ander materiaal en object - new THREE.Line(geom, new THREE.LineBasicMaterial({color: 0xffffff})) verbindt de punten in volgorde. Daarmee teken je banen, grafieken en draadsculpturen.

Oefeningen

Nabootsen

Pas bestaande code aan - verander één waarde en bekijk het resultaat.

Sneeuwkleur

Maak de punten wit, kleiner (size 0.02) en verdubbel het aantal. Andere wereld, drie getallen.

Variëren

Breid een voorbeeld uit met eigen aanpassingen.

Kubuswolk

Verdeel de punten over een kubus in plaats van een bol (gewoon drie keer Math.random, zoals in het eerste codeblok) en laat de wolk om twee assen draaien.

Creëren

Bouw iets nieuws met de concepten die je hebt geleerd.

Vallende sneeuw

Laat elk punt omlaag bewegen via de positie-array (met needsUpdate). Punt onder de vloer? Teleporteer het terug naar boven. Combineer met fog uit de vorige les voor een sneeuwstorm.

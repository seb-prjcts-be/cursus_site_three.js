Schaduwen zijn het verschil tussen vormen die zweven en een scene die echt voelt. Ze staan standaard uit omdat ze rekenkracht kosten - je zet ze aan op drie plekken: de renderer, het licht, en de objecten.

Kernpunten
  • Stap 1: renderer.shadowMap.enabled = true - de hoofdschakelaar
  • Stap 2: licht.castShadow = true - alleen Directional-, Point- en SpotLight kunnen het
  • Stap 3: per object castShadow (werpt schaduw) en/of receiveShadow (vangt schaduw)
  • Blokkerige randen? Vergroot de resolutie met licht.shadow.mapSize

De drie schakelaars

three.js laat je per onderdeel kiezen, want elke schaduw kost prestaties. Het volledige recept:

// 1. Renderer: hoofdschakelaar
renderer.shadowMap.enabled = true;
renderer.shadowMap.type = THREE.PCFSoftShadowMap;  // zachtere randen

// 2. Licht: dit licht werpt schaduwen
zon.castShadow = true;
zon.shadow.mapSize.set(2048, 2048);  // hogere resolutie = strakkere rand

// 3. Objecten: wie werpt, wie vangt
kubus.castShadow = true;      // de kubus werpt schaduw
vloer.receiveShadow = true;   // de vloer vangt ze op

Een object mag ook allebei: een bol op een tafel werpt schaduw op de tafel én vangt de schaduw van een lamp erboven.

Let op

AmbientLight kan geen schaduwen werpen - het komt per definitie van overal. Zie je geen schaduw, loop dan de drie schakelaars in volgorde na; negen van de tien keer ontbreekt er precies één.

Probeer het: stuiterende bol met schaduw

Kijk hoe de schaduw scherper wordt als de bol de vloer nadert en zachter als hij stijgt - dat doet PCFSoftShadowMap voor je. Zet de regel eens uit en vergelijk.

Als de schaduw er raar uitziet

  • Blokkerige rand - verhoog shadow.mapSize naar 2048 of 4096 (kost geheugen).
  • Schaduw afgekapt - bij DirectionalLight is het schaduwgebied een doos rond het nulpunt. Grote scene? Vergroot zon.shadow.camera.left/right/top/bottom.
  • Strepen op oppervlakken ("shadow acne") - speel met zon.shadow.bias, bijvoorbeeld -0.0005.
Weetje

Onder de motorkap rendert three.js de scene een extra keer vanuit het licht, naar een onzichtbare dieptetextuur (de shadow map). Daarom kosten schaduwen prestaties: elke schaduwwerpende lamp is een extra render per frame. p5 heeft dit mechanisme niet - dit is typisch het punt waar three.js de meerwaarde toont.

k.castShadow = true;
k.receiveShadow = true;

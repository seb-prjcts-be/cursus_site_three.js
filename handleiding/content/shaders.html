Shaders zijn kleine programma's die rechtstreeks op de videokaart draaien: één voor de vorm (vertex shader), één voor de kleur van elke pixel (fragment shader). Ken je shaders uit p5, dan is dit vertrouwd terrein in een nieuwe jas.

Kernpunten
  • ShaderMaterial is een materiaal waarvan jij de GLSL-code schrijft
  • Uniforms zijn variabelen die je vanuit JavaScript doorgeeft (tijd, muis, kleur)
  • three.js geeft je gratis position, normal en uv in de vertex shader
  • Zelfde GLSL-taal als p5-shaders, maar zonder eigen setUniform-gedoe: uniforms zijn een object

De kleinste shader

Een ShaderMaterial heeft twee stukken GLSL en een uniforms-object. Dit is het minimum dat iets toont:

const materiaal = new THREE.ShaderMaterial({
  uniforms: {
    uTijd: { value: 0 }          // vanuit JS bij te werken
  },
  vertexShader: `
    varying vec2 vUv;
    void main() {
      vUv = uv;                  // uv doorgeven aan de fragment shader
      gl_Position = projectionMatrix * modelViewMatrix * vec4(position, 1.0);
    }
  `,
  fragmentShader: `
    uniform float uTijd;
    varying vec2 vUv;
    void main() {
      gl_FragColor = vec4(vUv, abs(sin(uTijd)), 1.0);
    }
  `
});

In p5 deed loadShader() dit werk en gaf je uniforms door met setUniform. Hier is het uniforms-object de brug: materiaal.uniforms.uTijd.value = t in je loop, klaar.

De regel met projectionMatrix * modelViewMatrix is de standaard-regel die 3D-punten naar het scherm projecteert - die schrijf je in bijna elke vertex shader letterlijk zo over.

Let op

GLSL is streng: 1 is een int en 1.0 een float, en die mag je niet zomaar mengen. De foutmelding verschijnt in de console van je browser. Negen van de tien shader-bugs zijn een vergeten .0.

Wanneer grijp je naar een shader?

  • Effecten die per pixel of per vertex moeten gebeuren: golven, gradiënten, vervormingen, dissolves.
  • Wanneer duizenden objecten hetzelfde effect nodig hebben - de GPU doet het in één keer.
  • Niet voor dingen die een standaardmateriaal al kan: begin altijd bij MeshStandardMaterial en escaleer pas naar een shader als het echt moet.
Weetje

Elk ingebouwd materiaal van three.js ís een shader - MeshStandardMaterial is duizenden regels gegenereerde GLSL. Met ShaderMaterial schrijf je dus geen "extraatje", je neemt de pen over van de bibliotheek zelf. De nieuwste three.js-versies hebben daarnaast TSL (Three Shading Language) om shaders in JavaScript te schrijven voor WebGPU - vergelijkbaar met p5.strands in p5 2.x.

Muis als uniform

Voeg een uniform uMuis (Vector2) toe, vul hem vanuit een pointermove-listener, en laat de golfhoogte afhangen van de afstand tot de muis.

Hint
uniforms: { uTijd: {value: 0}, uMuis: {value: new THREE.Vector2()} }

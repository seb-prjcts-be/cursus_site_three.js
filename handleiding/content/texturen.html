Een textuur is een afbeelding die om een 3D-vorm gewikkeld wordt. De TextureLoader is de three.js-versie van loadImage() - en je plakt het resultaat rechtstreeks op een materiaal.

Kernpunten
  • new THREE.TextureLoader().load(pad) laadt een afbeelding als textuur
  • Je hangt de textuur aan het materiaal via de eigenschap map
  • Laden gaat asynchroon, maar three.js toont het object al en vult de textuur in zodra ze er is - geen preload() nodig
  • Elke geometrie heeft ingebouwde UV-coördinaten die bepalen hoe de afbeelding om de vorm wikkelt

Van loadImage naar TextureLoader

// p5
let img;
function preload() { img = loadImage('steen.jpg'); }

// three.js
const textuur = new THREE.TextureLoader().load('steen.jpg');
const materiaal = new THREE.MeshStandardMaterial({ map: textuur });

Geen preload-gedoe: de renderer tekent gewoon door en de textuur verschijnt zodra ze geladen is. De eigenschap heet map omdat er nog veel meer "maps" bestaan: roughnessMap, normalMap, bumpMap - allemaal afbeeldingen die een materiaaleigenschap per pixel sturen.

Probeer het: het UV-testraster

Dit voorbeeld laadt het officiële UV-testraster uit je lokale three.js-clone. Elk vakje toont waar dat stuk van de afbeelding op de vorm terechtkomt. Wissel de geometrie eens om: vervang de kubus door een SphereGeometry(1.5) of TorusKnotGeometry(1, 0.4).

De regel textuur.colorSpace = THREE.SRGBColorSpace hoort er bij kleurenafbeeldingen altijd bij. Vergeet je hem, dan ziet alles er net iets flets en verkeerd uit - subtiel genoeg om er lang naar te zoeken.

Weetje

UV-coördinaten heten zo omdat x en y al bezet waren door de 3D-ruimte. U loopt horizontaal door de afbeelding, V verticaal, beide van 0 tot 1. Het testraster met letters en cijfers bestaat precies om te zien welk (U, V)-vakje waar belandt.

Eigen afbeeldingen

Alles wat loadImage() kon laden, kan TextureLoader ook: foto's, je eigen tekeningen, zelfs een p5-export. Let op twee dingen:

  • De afbeelding moet via dezelfde server bereikbaar zijn (geen file://, en andermans servers blokkeren vaak - hetzelfde CORS-verhaal als bij p5).
  • Afmetingen in machten van twee (256, 512, 1024) renderen het efficiëntst.

In je lokale clone zit een hele map om mee te spelen: three.js/examples/textures/ - van baksteen tot lavasteen.

Hint
const mat = new THREE.MeshStandardMaterial({
  map: lader.load('../three.js/examples/textures/brick_diffuse.jpg'),
  bumpMap: lader.load('../three.js/examples/textures/brick_bump.jpg'),
  bumpScale: 1.5
});

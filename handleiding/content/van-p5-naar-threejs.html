Alles wat je in p5.js leerde, bestaat ook in three.js - het heet alleen anders en het werkt nét iets anders. Deze pagina is je woordenboek.

Kernpunten
  • p5 is immediate mode: elk frame teken je alles opnieuw
  • three.js is retained mode: objecten bestaan blijvend in een scene graph
  • In de animatieloop teken je niets - je wijzigt eigenschappen van bestaande objecten
  • De vertaaltabel hieronder koppelt elk p5-begrip aan zijn three.js-equivalent

Het grote verschil: tekenen versus bestaan

In p5 wist background() het canvas en teken je daarna alles opnieuw. Een cirkel bestaat alleen op het moment dat circle() wordt uitgevoerd:

// p5: de cirkel "bestaat" alleen tijdens dit commando
function draw() {
  background(240);
  circle(x, y, 60);
}

In three.js maak je objecten één keer aan. Ze blijven bestaan in de scene, en in de loop verander je alleen hun eigenschappen:

// three.js: de bol BESTAAT, elk frame verschuif je hem
const bol = new THREE.Mesh(geometry, material);
scene.add(bol);

renderer.setAnimationLoop(() => {
  bol.position.x += 0.01;       // eigenschap wijzigen
  renderer.render(scene, camera); // boom tekenen
});

Ken je classes in p5? Dan ken je dit denkpatroon al. Een class Ball met eigenschappen x en y die je in draw() bijwerkt - dat is precies hoe elk three.js-object werkt. three.js heeft dat patroon ingebouwd.

Valkuilen voor p5-handen

  1. Maak geen objecten in de loop

    new THREE.SphereGeometry() in de animatieloop is de three.js-versie van loadImage() in draw(): je geheugen loopt vol. Maak objecten één keer, wijzig daarna alleen eigenschappen.

  2. Y wijst omhoog

    Net als in p5 WEBGL-mode, maar anders dan in p5 2D. Positieve y is naar boven, en (0, 0, 0) ligt in het midden van de wereld.

  3. Kleuren zijn hex-getallen

    0xff007d in plaats van (255, 0, 125). Wil je toch p5-stijl schrijven, dan mag new THREE.Color('rgb(255, 0, 125)') ook.

  4. Hoeken zijn altijd radialen

    Er is geen angleMode(DEGREES). Gebruik THREE.MathUtils.degToRad(90) als je in graden wilt denken.

  5. Zonder licht is alles zwart

    Het standaard materiaal reageert op licht, zoals p5-vormen na lights(). Geen licht toegevoegd? Zwart object. Meer hierover bij Licht.
